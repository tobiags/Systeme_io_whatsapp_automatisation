# Questionnaire de Cadrage - Bot WhatsApp Challenge Amazon FBA

## 1. Objet du questionnaire

Ce document a pour but de cadrer précisément le comportement attendu du bot WhatsApp.

Il servira directement à configurer :

- le ton du bot
- les types de réponses autorisées
- les limites du bot
- les cas où le bot doit se taire
- les cas où le bot doit passer rapidement la main à un humain
- les relances automatiques acceptables
- les messages de suivi attendus avant, pendant et après le challenge

L'objectif est simple :

- éviter les réponses robotisées
- éviter les hallucinations
- éviter les réponses commerciales maladroites
- aligner le bot sur la réalité opérationnelle souhaitée par le client

---

## 2. Instructions de remplissage

- Répondez de façon concrète, même si la réponse est courte.
- Si plusieurs options conviennent, indiquez votre préférence principale.
- Si vous ne voulez pas que le bot fasse quelque chose, dites-le explicitement.
- Si vous préférez une intervention humaine sur certains sujets, dites-le aussi explicitement.

## 13. Résultat attendu après remplissage

À partir de ce questionnaire, le système sera ajusté sur 4 blocs :

- **Prompt principal** : ton, rôle, style, limites
- **Réponses locales** : messages approuvés, sobres, réutilisables
- **Guardrails** : ce qui est autorisé, interdit, ou soumis à escalade
- **Matrice bot / humain** : quand répondre, quand tagger, quand escalader

---

## 14. Note finale

Si une réponse du client est floue, elle doit être clarifiée avant implémentation.

Le but n'est pas d'avoir un bot qui parle beaucoup.

Le but est d'avoir un bot :

- utile
- crédible
- sobre
- cadré
- productif
- et conforme à la manière dont le client veut réellement gérer ses leads.
